What powers InfraMonitor

Every layer of the stack — from image decoding to health scoring — is built on open-source libraries. Here's what each one does and why we chose it.

AI / Machine Learning
🤖

Ultralytics YOLOv11

ultralytics >= 8.3.0

YOLO (You Only Look Once) is a real-time object detection and segmentation architecture. We use the YOLOv11 nano segmentation variant — the smallest and fastest in the family — which outputs pixel-level instance masks rather than just bounding boxes. This lets us know exactly which pixels are crack, not just which rectangle contains one.

Used for: crack segmentation mask · seep detection mask
📦

ONNX Runtime

onnxruntime >= 1.17.0

An inference engine for running models exported to the Open Neural Network Exchange format. ONNX Runtime allows models trained in any framework to run efficiently on CPU without the full training library installed. Used as a secondary inference backend when Ultralytics is unavailable.

Used for: fallback model inference on CPU
Computer Vision
👁️

OpenCV

opencv-python >= 4.8.0

The industry-standard computer vision library. We use it for image decoding, colour space conversion (BGR → HSV → Gray), CLAHE contrast enhancement, Canny edge detection, morphological operations (dilation, closing, erosion), connected component labelling, heatmap generation with COLORMAP_JET, and drawing annotations on output images.

Used for: almost everything image-related
🔬

scikit-image

scikit-image >= 0.21.0

A scientific image processing library built on NumPy. We use its morphology.skeletonize() to thin crack masks to single-pixel-wide lines (removing thick blobs while preserving crack topology), and measure.regionprops() to compute per-region statistics like area, perimeter, centroid, and bounding box — used in false positive filtering and crack width estimation.

Used for: skeletonisation · region property measurement
📐

SciPy (ndimage)

scipy >= 1.11.0

SciPy's ndimage module provides multi-dimensional image processing primitives. We use ndimage.label() to assign unique integer labels to each connected crack region, and ndimage.sum() to measure the length of each skeleton segment so short noise fragments can be filtered out.

Used for: connected component labelling · skeleton length filtering
Data & Numerics
🔢

NumPy

numpy >= 1.24.0

The foundation of scientific Python. Every image in our pipeline is a NumPy array (height × width × channels). We use NumPy for float32 image arithmetic, mask creation via boolean indexing, bitwise operations, weighted blending of AI and OpenCV masks, and statistical operations like mean and clip for score calculation.

Used for: all array operations throughout the pipeline
🖼️

Pillow

Pillow >= 10.0.0

Python's imaging library, used for reading image formats that OpenCV handles less gracefully (such as WEBP and certain PNG variants) and for any image format conversion tasks during file handling in the Django upload pipeline.

Used for: image format compatibility · file handling
Web Framework & Infrastructure
🐍

Django

django >= 4.2

The backend web framework. Django handles HTTP routing, CSRF protection, session management (for passing scan results to the report page), the ORM (Object Relational Mapper) for storing scan history in SQLite, static file serving, and the template engine that renders the HTML pages you're reading now.

Used for: routing · sessions · database ORM · templates · file serving
🗄️

PostgreSQL via Supabase

supabase.com

A managed PostgreSQL database hosted on Supabase. Every scan result — health score, sub-scores, detection stats, AI confidence, warnings, and recommendations — is stored as a ScanResult row, along with user accounts and credentials, accessed through Django's ORM. Supabase was chosen over a self-hosted database because it stays available continuously, unlike ephemeral storage on free-tier hosting. it sends the url of the uploaded image to the backend for processing, and the backend returns the scan results which are then displayed on the report page.

Used for: persistent scan history · user accounts · production database
